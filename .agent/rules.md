# Workspace Rules
1. 常に `docs/development_workflow.md` を最優先の指示書として遵守し、全ての作業前にこのファイルの内容と現在のタスクの整合性を確認してください。
2. ドキュメント（Plan/Task等）は必ず `docs/` ディレクトリ配下で管理してください。
3. Plan / Task に記載された目的・スコープ・制約は、人間の承認なしに変更してはなりません。変更が必要な場合は「変更提案」として明示してください。
4. 実装作業を開始する前に、対応する Plan / Task が docs/ 配下に存在することを必ず確認してください。
5. 指示や仕様に曖昧さや矛盾を感じた場合、自己判断で補完せず、必ず質問または確認を行ってください。
